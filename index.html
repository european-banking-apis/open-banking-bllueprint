My Name is Kin Lane, this is my Open Banking API Portal Template

Open Banking is doing interesting work in the UK when it comes to banking APIs. They have produced a set of banking API specifications, along with a directory, dispute and communication management, which is really moving forward the banking API conversation in the UK.

I have been learning more about the Open Banking API specification, and would like to get the word out about their work, and hopefully contribute to the conversation, as well as tooling available when it comes to banking APIs in the UK.

This project is meant to be a forkable project that any bank in the UK, or anywhere else for that matter, publish, change, and turn into their own API portal. This project is a work in progress, and I will be releasing more features soon.

Getting Started

Here is a quick list of the eight services, which there are separate documentation available below:

  • Read / Write Accounts - The read/write APIs for managing accounts.
  • Read / Write Payments - The read/write APIs for managing payments.
  • Public Data - ATMs - The public data for bank ATMs.
  • Public Branches - ATMs - The public data for bank ATMs.
  • Public Data - Personal Accounts - The public data for bank personal accounts.
  • Public Data - Business Accounts - The public data for bank business accounts.
  • Public Data - SME Unsecured Loans - The public data for bank SME unsecured loans.
  • Public Data - Commercial Credit Cards - The public data for bank commercial credit cards.

This isn't a live API, or mock API, it is mean for presentation purposes at this moment--more to come.

Read / Write APIs

This is the heart of the Open Banking API specification, providing detailed read and write APIs for both accounts and payment APIs. Providing the most value to the personal and business API customers, as well as other key stakeholders.

Accounts

This is a read / write API for accessing all the details of a consumer or banking account.

This is just an OpenAPI definition, published using Swagger UI--it is not a functional API or mock yet.

Payments

This is a read / write API for payments, and payment submissionfs for a consumer or banking account.

This is just an OpenAPI definition, published using Swagger UI--it is not a functional API or mock yet.

Public Data APIs

These are the APIs which provie access to the public profile, products, and services for the banks. Allowing access to some of the data that is commonly scraped for use in other 3rd party applications.

This section addresses wider appetite for data about each bank, making it so the common information about banks are accessible in a standardized, machine readable way across all banks in the UK.

Support

There are three ways that you can get support when using the API--consider one of the following channels when you have a question, comment, or some feedback regarding the API and integration.

Github Twitter Email

It is recommended that you start with submitting a Github issue, then maybe tweet at us. If your questions or comments require some privacy you should use email. Make sure you look through the existing issues before submitting a new one.
